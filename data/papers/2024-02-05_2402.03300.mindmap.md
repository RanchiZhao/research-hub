# DeepSeekMath / GRPO

## Problem: PPO's Critic Bottleneck
- PPO requires a value function model of comparable size to the policy
- For 7B policy, critic is another ~7B model — doubling memory
- Per-token value estimation is ill-defined for LLMs

## GRPO Core Idea: Group-Relative Baseline
- Eliminate the critic entirely
- For each question, sample G outputs from current policy
- Score each with reward model → {r_1, ..., r_G}
- Compute group-relative advantages: A_i = (r_i - mean(r)) / std(r)
- Group mean acts as baseline (replacing critic)

## Mathematical Formulation
- Clipped surrogate objective (same as PPO)
  - min[ratio * A, clip(ratio, 1-ε, 1+ε) * A]
- KL regularization added directly to loss
  - Not as reward penalty
  - Controlled by coefficient β

## Two Supervision Modes
- **Outcome Supervision (OS)**
  - Single reward per complete output
  - All tokens share same advantage
- **Process Supervision (PS)**
  - Per-step rewards
  - Finer-grained signal → better performance

## Unified Paradigm
- RFT, DPO, PPO, GRPO as gradient variants
- Three axes: data source, reward function, gradient coefficient
- Online > Offline (gap widens with training)

## Results
- MATH: 46.8% (SFT) → 51.7% (GRPO)
- GSM8K: 82.9% (SFT) → 88.2% (GRPO)
- Memory: ~50% reduction vs PPO

## Limitations
- Geometry and theorem-proving remain weak
- Reward model has ~20% error rate
