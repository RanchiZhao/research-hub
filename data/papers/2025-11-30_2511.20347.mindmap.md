# SAPO: Soft Adaptive Policy Optimization

## Problem: Why Hard Clipping Fails
- GRPO's hard token clip: gradient is either full or ZERO
- GSPO's hard sequence clip: one bad token kills entire sequence gradient
- Both create discontinuous optimization landscapes

## Core Innovation: Soft Gating
- f(x) = σ(τ(x-1)) * 4/τ
  - x = importance ratio r_t
  - τ = temperature parameter
  - Continuous, smooth, maximum at on-policy point
  - Never fully zeros out gradient

## Asymmetric Temperature Design
- Negative advantage updates → higher variance, more unstable
- Positive advantage updates → lower variance, more stable
- τ_neg > τ_pos (narrower gate for negative updates)
- Prevents training collapse from destabilizing negative gradients

## Comparison
- vs GRPO: soft sigmoid gate → hard token clip
- vs GSPO: selective per token → all-or-nothing per sequence
- Best of both: sequence-coherent + token-adaptive

## Results
- Cold-start Qwen3-30B-A3B: stable learning where GSPO/GRPO collapse
- Qwen3-VL: consistent gains across model sizes
- Powers Qwen3-VL model series

## Practical Implications
- Drop-in replacement for clipped surrogate objective
- Eliminates need for routing replay stabilization (MoE)
- Single smooth function replaces piecewise logic
