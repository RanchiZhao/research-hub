# CISPO: Clipped IS-weight Policy Optimization (MiniMax-M1)

## Problem: Token-Level Clipping Kills Reasoning Tokens
- PPO/GRPO clip token-level ratio: clip(r_t × Â, 1-ε, 1+ε)
- Low-probability tokens ("However", "Recheck") are rare in base model
- After first update, these get high r_t → clipped out entirely
- Long CoT reasoning behaviors cannot emerge

## Core Idea: Clip the Weight, Not the Update
- Move clipping from token updates to importance sampling weights
- J = E[sg(clip(r_t)) × Â × log π_θ]
- sg() = stop-gradient: IS weight is pure scalar multiplier
- All tokens always contribute gradients

## Why It Works
- PPO/GRPO: Gradient = 0 when r_t outside clip range
- DAPO: Raises upper ε → partial mitigation, still clips eventually
- CISPO: Gradient ≠ 0 for any token (log π_θ always active)
- Result: 2x sample efficiency vs DAPO

## Engineering Insights
- FP32 for LM output head: fixes training-inference probability mismatch
- Adam eps = 1e-15 (not 1e-8): gradients span 1e-18 to 1e-5
- Early truncation: halt if 3000 tokens each have p > 0.99

## Curriculum Strategy
- Phase 1: Reasoning-only (math, logic, code)
- Phase 2: Mixed general domain
- Phase 3: Length scaling (40K → 80K thinking budget)

## Results (MiniMax-M1)
- AIME 2024: 86.0%, MATH-500: 96.8%
- 456B total params, 45.9B active (MoE)
- Native 1M token context
- 3 weeks, 512 H800 GPUs
