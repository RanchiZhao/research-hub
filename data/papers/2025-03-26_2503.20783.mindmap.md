# Dr. GRPO: Understanding R1-Zero-Like Training

## GRPO's Two Bugs
### Bias #1: Length Bias
- GRPO divides advantage by response length |o_i|
- Correct responses: shorter gets larger per-token gradient
- Incorrect responses: longer gets smaller penalty
- Result: incorrect responses grow unboundedly during training

### Bias #2: Difficulty Bias
- GRPO normalizes by std(rewards) within each group
- Easy questions (most correct): small std → huge gradient
- Hard questions (most wrong): small std → huge gradient
- Medium questions (where learning happens): normal gradient
- Result: extreme-difficulty questions dominate

## Dr. GRPO: The Fix
- Remove (1/|o_i|) length normalization
- Remove std(r) normalization
- What remains: clean REINFORCE with leave-one-out baseline (RLOO)
- A_i = r_i - mean(r_{-i})

## Observable Effects
- Response length: controlled growth (no unbounded inflation)
- Token efficiency: fewer wasted tokens on verbose wrong answers
- Gradient balance: all difficulty levels contribute proportionally

## Base Model Findings
- Aha moments already exist in DeepSeek-V3-Base before any RL
- Qwen2.5 without template >> Qwen2.5 with R1 template (~60% better)
- RL amplifies, not creates, reasoning behaviors

## Results
- AIME 2024: 43.3% (Qwen2.5-7B)
- MATH500: 80.0% (Qwen2.5-7B)
- Training: 8x A100, ~27 hours

## Practical Takeaway
- Check your GRPO/PPO for length normalization — remove it
- Remove std normalization of advantages
- Two lines of code to delete, meaningful improvement
