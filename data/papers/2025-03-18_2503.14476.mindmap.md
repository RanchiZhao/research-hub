# DAPO: An Open-Source LLM Reinforcement Learning System at Scale

## Problem
- Naive GRPO suffers from entropy collapse — model converges to narrow policy distributions too early
- Prompts where all responses are correct or incorrect produce zero effective gradients
- Sample-level loss normalization dilutes learning signal from longer reasoning chains
- Hard truncation of overlong responses introduces noisy reward signals

## Method
- **Clip-Higher (Decoupled Clipping)**
  - Separate lower bound (ε_low=0.2) and upper bound (ε_high=0.28)
  - Prevents entropy collapse by allowing more upward exploration
- **Dynamic Sampling**
  - Filter out prompts where accuracy = 0 or 1
  - Ensures every prompt contributes meaningful gradient signal
- **Token-Level Policy Gradient Loss**
  - Normalize loss by total token count across batch
  - Longer reasoning chains get proportionally more weight
- **Overlong Reward Shaping**
  - Soft graduated punishment based on response length
  - Reduces noise from conflating verbose-but-correct with truncated

## Experiments
- Base model: Qwen2.5-32B
- Dataset: DAPO-Math-17K
- Reward: Binary rule-based (±1)
- Main result: 50 points AIME 2024 (vs R1-Zero's 47, 50% fewer steps)
- Ablation: 30 → 36 → 38 → 41 → 42 → 50

## Innovation
- First to decouple PPO clipping bounds asymmetrically for LLM RL
- Dynamic sampling at prompt level is simple but effective
- Complete open-source release: algorithm + infra (verl) + dataset

## Limitations
- Focused exclusively on mathematical reasoning
- Rule-based binary reward only works for verifiable domains
- Training reward correlates poorly with validation accuracy
